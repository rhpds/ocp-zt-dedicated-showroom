= Module 3: As Developer User (Testing send-to-tty)

This module demonstrates the *Send-To tty terminal* pattern. Code blocks use `role="send-to-tty"` — click ▶ to send commands to the *Terminal (tty)* tab.

NOTE: The tty tab requires a ttyd-based terminal at `/tty`. The `execute-listener.js` is injected automatically by the chart. If your lab uses wetty only, use `send-to-wetty` instead.

Create a Secret in `demo-project` as the `developer` user.

== Credentials

[cols="1,2"]
|===
| OCP API URL | `{openshift_api_server_url}`
| Username    | `developer`
| Password    | `{openshift_cluster_admin_password}`
|===

== Your Task

Use the *Terminal (tty)* tab — click ▶:

[source,sh,role="send-to-tty",subs="attributes+"]
----
oc login {openshift_api_server_url} \
  -u developer \
  -p {openshift_cluster_admin_password} \
  --insecure-skip-tls-verify

oc create secret generic dev-secret \
  --from-literal=api-key=my-app-key \
  -n demo-project
----

== Verify

Click *Validate* to check that `dev-secret` exists in `demo-project` and is accessible by `developer`.

Click *Solve* to have the grader create it using `oc --as developer`.
