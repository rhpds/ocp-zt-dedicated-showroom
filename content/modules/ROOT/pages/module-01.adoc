= Module 1: Admin Task (Testing send-to-wetty)

This module demonstrates the *Send-To wetty terminal* pattern. Code blocks use `role="send-to-terminal"` — click ▶ to send commands to the *Terminal* tab.

Create a namespace `demo-project` and grant the `developer` user admin access.

== Credentials

[cols="1,2"]
|===
| OCP API URL | `{openshift_api_server_url}`
| Username    | `{openshift_cluster_admin_username}`
| Password    | `{openshift_cluster_admin_password}`
| Console     | `{openshift_cluster_console_url}`
|===

== Your Task

Use the *Terminal* tab — click ▶ to send directly:

[source,sh,role="send-to-terminal",subs="attributes+"]
----
oc login {openshift_api_server_url} \
  -u {openshift_cluster_admin_username} \
  -p {openshift_cluster_admin_password} \
  --insecure-skip-tls-verify

oc new-project demo-project
oc adm policy add-role-to-user admin developer -n demo-project
----

== Verify

Click *Validate* to check that `demo-project` exists and `developer` has admin access.

Click *Solve* to have the grader complete this for you.
